The Domain Model Language

The domain model language is a simple language to model entities and there relations textually. Originally, it compiles POJOs creating fields, getters and setters for each feature declared in an entity. We extended the compiler to also support validation and server side auto-generation of values.

A small code snippet of a simple entity is given below.

package app {
    entity Message {
        text: String
        generated created : String
    }
    generate {
        if(this.getCreated == null)
            this.setCreated(new Date().toString)
    }
}

An entitiy Message is declared in the package app.model, containing two String type features. One of these features is marked to be generated. This is done by calling the generate block. The generate block can contain arbitrary Xbase expressions. The above entity compiles to the following Java code.

...
@Entity
@XmlRootElement
public class Message {
  @GeneratedValue
  @Id
  private int id;
  
  public int getId() {
    return this.id;
  }
  
  public void setId(final int id) {
    this.id = id;
  }
  
  private String text;
  
  public String getText() {
    return this.text;
  }
  
  public void setText(final String text) {
    this.text = text;
  }
  
  private String created;
  
  public String getCreated() {
    return this.created;
  }
  
  public void setCreated(final String created) {
    this.created = created;
  }
  
  public void generate() {
    String _created = this.getCreated();
    boolean _equals = ObjectExtensions.operator_equals(_created, null);
    if (_equals) {
      Date _date = new Date();
      String _string = _date.toString();
      this.setCreated(_string);
    }
  }
}

As you can see, it is plain java code with some annotations. For each property in the declared entity a getter and a setter is generated. Also, if the entity did not declare a field named id, it is added automatically. The class annotation @Entity declares the class to be a persistable entity using the Java Persistence Api, while the @XmlRootElement (a JAXB annotation) annotation denotes that the entity is serializable to XML.

So the DAO class encapsulates the whole database access logics to plain Java methods, implementing the typical CRUD operations. This API can be reused by any Java language such as the service language described in Service.
